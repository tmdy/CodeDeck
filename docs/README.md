## 维护规则

- README 面向首次接手项目的人，只写项目用途、结构、命令和关键运行数据。
- `docs/specs/` 面向实现和维护，只保留仍能指导当前代码的内容。
- 同一主题不要拆成“规格 / 实现 / 测试说明”多份重复文档；优先合并为一份按目标、模型、实现、限制、验证组织的文档。
- 过时设计如果仍有历史价值，应在标题或开头说明状态；没有维护价值的文档直接删除。
